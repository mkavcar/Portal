Pacing Report Details

clear

Pacing reports allow media planning teams to monitor the rate of trafficking against campaign plans. The primary goal of these reports is to indicate whether the campaign's budget is being spent evenly across the allotted time period. Multiple measures are provided as lenses into this how delivery unfolds for a given campaign, on a raw, running sum, and percentage basis. Some of the measures are as follows:

  • Days completed and remaining in a given campaign
  • Impressions delivered vs planned
  • Actual capped and uncapped cost vs planned

Please note, all the reports have data available from 1/1/2015 to the prior day except for Moat whose data is available only from 5/1/2016.

 

Report Data Sources

CCR uses the following data sources:

Campaign Management: Prisma

Primary Ad Servers: DFA & Sizmek

Campaign Verification: DoubleVerify & Moat

 

Report Types

Due to the varying level of granularity of the reports, CCR provides these reports in four categories:

 

Summary Reports

We provide summary reports for a high-level view of how each client’s trafficking is pacing by campaign, inventory type, and site. Because CCR’s data can be very dense at its most granular level, these reports provide superior performance when seeking to access it.

These reports include:

  • Overall Pacing
  • A report which gives an overall dashboard view of pacing of all campaigns across multiple clients you have access to. Click here for report definition.
  • Pacing by Campaign
  • A report which tracks pacing for all campaigns. Click here for report definition.
  • Pacing by Inventory Type
  • A report which tracks pacing for all campaigns by the Inventory Type they were trafficked on. Click here for report definition.
  • Pacing by Site
  • A report which tracks pacing for all campaigns by the Sites they were trafficked on. Click here for report definition.

 

Detailed Reports – By Delivery Period

This category of reports provides campaign pacing details on varying levels of delivery periods, namely, Monthly and Daily.

  • Pacing by Month
  • A monthly report which tracks pacing for all campaigns by the Sites they were trafficked on. Click here for report definition.
  • Running Sum by Month
  • A running sum report which tracks pacing for all campaigns by the Sites they were trafficked on. Click here for report definition.
  • Pacing by Date
  • A daily report which tracks pacing for all campaigns by the Sites they were trafficked on. Click here for report definition.
  • Running Sum by Date
  • A running sum report which tracks pacing for all campaigns by the Sites they were trafficked on. Click here for report definition.

 

Detailed Reports – By Granularity

This category of reports provides campaign pacing details on varying levels granularity, namely, Site, Creative and Placement.

  • Pacing by Site and Creative
  • A report which tracks pacing for all campaigns by the Sites they were trafficked on and the Creatives which were trafficked. Click here for report definition.
  • Pacing by Creative and Site
  • A report which tracks pacing for all campaigns by the Creatives which were trafficked and the Sites they were trafficked on. Click here for report definition.
  • Pacing by Placement
  • A report which tracks pacing for all campaigns by the planned placements. Click here for report definition.

 

Detailed Reports – Placement by Date

This category of reports provides campaign pacing details on the most granular daily level.

  • Pacing by Placement by Date
  • This is the most granular pacing report which tracks pacing for all campaign by the planned placement on a daily level. Click here for report definition.

 

Working with the reports - Downloaded Report Files

This section covers some important information you must know about the downloaded reports.

File Format

All report files downloaded from the CCR website are Excel Files. When the user opens the files for the first time, they will need to click on the “Enable Content” button that appears on top of the file.

Note: Make sure you have Excel 2013 to use these reports. Some functionality may not work if you have a lower version of Excel.

Note: On clicking the “Enable Content” button, the file will automatically refresh to display the client data the user has access to. For this automatic refresh to take place, make sure you’re on GroupM network or VPN. Please allow a moment for the report to fetch all of your data.

Note: Make sure you have approved access to the data.

 

Report Filters Worksheet

“Refresh Data” Button

The “Refresh Data” button helps you to refresh the reports to the latest available data. Clicking on this button will not reset your filters. The data is processed on a daily basis, and the latest data available at any time is 48 hours prior to the current date at launch. Overtime, we will aim to reduce this to a 24 hour window.

“Clear Filters” Button

The “Clear Filters” button resets all selected filters.

 

Data Filter Parameters - Summary Reports

Both Pacing and Performance Summary reports have the same filter parameters. Details of these parameters are given below:

  • Country
  • This filter allows you to filter your clients by the country you manage. If you have access to clients in both US and Canada, this filter will show both buttons, otherwise, it will just show the one country your clients reside.
  • Calendar Year
  • CCR contains Planned and Delivered for full year 2015 onwards, this filter allows you to view data for either one year or both years.
  • Client
  • This filter will contain a list of all clients you have access to. Select single or multiple clients depending on what client’s data you wish to view in the reports.

 

Data Filter Parameters - Detailed Reports

Both Pacing and Performance Detailed reports have the same filter parameters. Details of these parameters are given below:

  • Country
  • This filter allows you to filter your clients by the country you manage. If you have access to clients in both US and Canada, this filter will show both buttons, otherwise, it will just show the one country your clients reside.
  • Date Range
  • Since these reports are more detailed, we have provided a date slider, which allows you to select monthly date ranges instead of just years. This will allow you to filter data further by days, months and quarters. Select a start and end date by expanding the selected portion of the slider across a date range. This filter allows you to view your data as detailed as a day to as high level as a year. The slider itself can be changed to select days, months, quarters or years.
  • Client
  • This filter will contain a list of all clients you have access to. Select single or multiple clients depending on what client’s data you wish to view in the reports.

 

Report Definitions

Overall Pacing

Dimension Name Description
Client Client name
Product Product name
Campaign Name The Campaign Name
Campaign Start Date The campaign start date
Campaign End Date The campaign end date
Metric Name Description
% Days Completed (Delivered) Displays the percentage of days into the campaign. [days into the campaign / total days of campaign]
% Days Left Displays the percentage of days remaining in the campaign. [days left from campaign / total days of campaign]
Run Rate [actual capped cost / planned cost * number of days of campaign / number days into campaign]
Planned Impressions

Number of impressions the campaign is scheduled to achieve.

Note: Planned Impressions may not always be established in Prisma by the media planning team, so it's possible to see delivered impressions where no impressions are planned.

Actual Impressions Number of impressions delivered for the campaign. This value is always uncapped.
Planned Impressions Left Number of planned impressions left. [planned impressions - actual impressions]

% Impressions Delivered

("Impressions Pacing")

Percentage of planned impressions delivered. [actual impressions / planned impressions]
% Impressions Left Percentage of planned impressions left. [planned impressions left/ planned impressions]
Payable Impressions The number of impressions that are considered payable as per the GroupM viewability guidelines. They are the human viewable impression counts, include the scaling factor for non-measured, and accommodates the GroupM rules for Large Ad Units and Multi-Ad Units.
Measurable Impressions The number of impressions where it was possible to determine whether the ad surface was 100% on-screen and were determined to be delivered to a human end point. These are Fraud free impressions where measurement can be attempted.
Viewable Impressions The number of impressions where it was possible to determine whether the ad surface was 100% on-screen and were determined to be delivered to a human end point
% Measurable [Measurable Impressions]/ ([Measurable Impressions] + [NonMeasuredImpressions])
% Viewable [Viewable Impressions] / [Measurable Impressions]
Planned Cost Total planned spend for the campaign.

Actual Capped Cost

Capping methods:

Date Capped:

Cost is "date capped", meaning that no spend is reported outside of planned flight dates.

Cost Capped:

This occurs at the package and flight level. Which means:

  1. Capping will only occur when the budget for a package over the duration of its entire flight is exceeded.
  2. Levels of aggregation higher than package may show Actual Uncapped Cost exceeding Planned Cost, while Actual Capped Cost is lower than Planned Cost, if some packages over-deliver while others under-deliver.
  3. Placements in a package may show arbitrary over-delivery and under-delivery, while the entire package’s capped cost will not exceed the plan.

Because capping occurs at a package level, which may add complexity at lower or higher levels of aggregation, the most reliable way to monitor pacing is via the running sum reports.
Actual Uncapped Cost Total actual spend reported for the campaign.
Planned Cost Left Planned amount left in the campaign. [planned cost - actual capped cost]

% Actual Cost Delivered

("Budget Pacing")

Percentage of planned cost spent. [actual capped cost / planned cost]
% Planned Cost Left Percentage of planned cost left in the campaign. [planned cost left / planned cost]

Actual Capped Cost

Capping methods:

Date Capped:

Cost is "date capped", meaning that no spend is reported outside of planned flight dates.

Cost Capped:

This occurs at the package and flight level. Which means:

  1. Capping will only occur when the budget for a package over the duration of its entire flight is exceeded.
  2. Levels of aggregation higher than package may show Actual Uncapped Cost exceeding Planned Cost, while Actual Capped Cost is lower than Planned Cost, if some packages over-deliver while others under-deliver.
  3. Placements in a package may show arbitrary over-delivery and under-delivery, while the entire package’s capped cost will not exceed the plan.

Because capping occurs at a package level, which may add complexity at lower or higher levels of aggregation, the most reliable way to monitor pacing is via the running sum reports.
